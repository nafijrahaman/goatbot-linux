Setup Commands

STEP 01

Enter Ubuntu

Log into the Ubuntu proot environment to begin managing your bot.

proot-distro login ubuntu
STEP 02

Go to Bot Directory

Navigate into your bot's working folder where all scripts and configs are located.

cd ~/bot
STEP 03

Sync Updated Files

Copy updated account credentials, config, and environment files from your device storage into the bot directory.

cp /storage/emulated/0/bot/{account.txt,account.dev.txt,.env} ~/bot/
STEP 04

Run Bot

Start the bot in the foreground to see live output and debug logs in the terminal.

node index.js
STEP 05

Keep Bot Alive — nohup

Run the bot in the background using nohup so it keeps running even after closing the terminal session.

nohup node index.js &
STEP 06

Check Running Processes

List all active Node.js processes to verify your bot is running and find its process ID.

ps aux | grep node
STEP 07

Stop Bot

Terminate all running Node.js bot processes immediately. Use with caution in production.

pkill node

PM2 — Process Manager

PM2 — 01

Install PM2 & Start Bot

Install PM2 globally and launch the bot with automatic restart on crash — the recommended production setup.

npm install -g pm2 && pm2 start index.js --name bot
PM2 — 02

Restart Bot

Gracefully restart the bot managed by PM2 without losing the process configuration.

pm2 restart bot
PM2 — 03

View Live Logs

Stream real-time logs of your bot's activity, errors, and output directly in the terminal.

pm2 logs bot
PM2 — 04

Stop Bot (PM2)

Stop the bot process managed by PM2 while keeping it listed in the process manager.

pm2 stop bot
PM2 — 05

List All Processes

Display a status table of all PM2-managed processes including uptime, memory, and CPU usage.

pm2 list
PM2 — 06

Save & Enable Auto-Start

Save the current PM2 process list and generate a startup script so the bot auto-starts on system reboot.

pm2 save && pm2 startup
